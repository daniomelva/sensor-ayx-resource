# sensor-ayx-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不依赖任何特定后端服务，可以直接通过浏览器打开使用。

## 项目简介

sensor-ayx-resource 是一个静态资源仓库，主要存放各类独立功能的 HTML 文件。每个页面均为自包含的完整页面，适合作为工具页面、信息展示或轻量级交互界面使用。

## 目录结构

```
.
├── pages/          # HTML 页面存放目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/`：所有 HTML 页面文件均放置在此目录下，每个文件代表一个独立功能页面。
- `assets/`：存放页面所需的公共或专用资源文件，按需组织。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，不依赖外部服务或数据库。
- 页面之间无耦合关系，可以单独使用或部署。
- 归档的页面可能包含 JavaScript 交互逻辑、CSS 样式和嵌入的媒体资源。
- 建议使用现代浏览器（如 Chrome、Firefox、Edge 等）打开查看。

## 维护说明

- 本仓库仅作为静态资源归档，不提供实时更新或动态内容。
- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并确保所有资源引用路径正确。
- 建议在新增或修改页面时，保持文件命名清晰、内容结构整洁。
- 不推荐在页面中引入外部不可控资源，以保持长期可用性。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 也可将整个仓库部署到任意静态文件服务器（如 GitHub Pages、Nginx 等）进行访问。

## 许可证

本仓库内容仅供学习与参考使用，不设特定许可证限制。如有特殊需求，请参考各页面内的说明。
